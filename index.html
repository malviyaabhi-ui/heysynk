Why context-aware AI changes everything for customer support — and how we built it

Most AI chatbots are glorified keyword matchers. heySynk is different. Here's exactly how Mira AI reads a full conversation and drafts a reply that actually makes sense.

AM
Alex Morgan
Head of Product at heySynk

Imagine a customer writes: "I need to return this." A keyword-based chatbot sees "return" and fires a generic returns policy link. But what if this customer messaged you three days ago about a broken product, your agent already promised a replacement, and now they're following up? The keyword bot has no idea. It sends the policy anyway. The customer is frustrated.

This is the fundamental failure of rule-based AI in customer support. And it's exactly what heySynk was built to solve.

How Mira AI reads a conversation

When an agent clicks "AI Reply" in heySynk, we don't just send the last message to the AI model. We send the entire conversation thread, the customer's profile, their previous conversation history, and the relevant articles from your Knowledge Base — all in a single structured prompt.

What we send to Mira AI

Full message history · Customer profile & tags · Previous conversation summaries · Matched KB articles · Agent's display name · Workspace tone guidelines

Mira AI then generates a reply that: references what's already been said (no repetition), applies relevant policies from your KB, matches the established tone of the thread, and addresses the specific concern — not a generic version of it.

The results in practice

Across heySynk workspaces, agents edit fewer than 20% of AI-generated replies before sending. The average time from opening a conversation to sending a reply dropped from 4 minutes to under 45 seconds for teams that use AI Reply consistently.

Why agents stay in control

Every AI draft is editable before it's sent. We made a deliberate product decision: AI should never send without explicit agent approval. This isn't a technical limitation — it's intentional. The agent reads the draft, edits if needed, and decides to send. This keeps quality high and prevents the kind of automated errors that destroy customer trust.

The goal is to make each agent dramatically more efficient — handling 3–5× more conversations without sacrificing quality — not to remove them from the equation.

Introduction to heySynk

What is heySynk?

heySynk is a multi-tenant AI-powered customer support platform. It provides a shared inbox, knowledge base, CRM, live visitor intelligence, campaigns, and analytics — all in one product.

The REST API lets you build integrations, automate workflows, sync data with your existing tools, and embed the chat widget on any website.

📍 Base URL: https://api.heysynk.app/v1 — All API requests use this endpoint with your API key.

Architecture

heySynk is built multi-tenant from the ground up. All data is scoped to a workspace. Each workspace has its own agents, conversations, contacts, settings, and billing.

Core concepts

  • Workspace — A team or company account. All data is isolated per workspace.
  • Agent — A team member with a role and defined permissions.
  • Conversation — A thread between a customer and agent(s), across any channel.
  • Contact — A customer record with history, notes, and tags.
  • Article — A Knowledge Base entry, optionally published to the public Help Centre.
  • Campaign — A rule-based automated message triggered by visitor behaviour.

Rate limits

API requests are rate limited per workspace. Default limits:

  • Free plan: 60 requests/minute
  • Starter: 300 requests/minute
  • Pro: 1,000 requests/minute
  • Enterprise: Custom

When you hit a rate limit you'll receive a 429 Too Many Requests response with a Retry-After header.

Quick Start

Get heySynk running in under 5 minutes.

Step 1 — Get your API key

Log in to app.heysynk.app → Admin Panel → API Keys → Generate Key. Copy and store it securely — it won't be shown again.

Step 2 — Embed the chat widget

Add this one line before the closing </body> tag on your website:

<script
  src="https://cdn.heysynk.app/widget.js"
  data-workspace="YOUR_WORKSPACE_SLUG"
  data-color="#2563eb"
  data-position="br"
></script>

Step 3 — Make your first API call

curl https://api.heysynk.app/v1/conversations   -H "Authorization: Bearer YOUR_API_KEY"   -H "Content-Type: application/json"

You'll get back a list of conversations in your workspace.

Step 4 — Create a contact

curl -X POST https://api.heysynk.app/v1/contacts   -H "Authorization: Bearer YOUR_API_KEY"   -H "Content-Type: application/json"   -d '{
    "name": "Jane Smith",
    "email": "jane@company.com",
    "company": "Acme Corp"
  }'

✓ That's it. Your widget is live, your first API call works. Check the sidebar for full API reference.

Authentication

All API requests must include your API key as a Bearer token.

API Keys

Generate your API key from Admin Panel → API Keys. Each key is scoped to a workspace and has an optional permission set.

Authorization: Bearer hsk_live_xxxxxxxxxxxxxxxxxxxx

Key types

  • Live keys — hsk_live_ prefix. Use in production.
  • Test keys — hsk_test_ prefix. Use in development. No real emails sent.

Security

  • Never expose API keys in client-side code or public repos
  • Use environment variables to store keys
  • Rotate keys immediately if compromised — Admin Panel → API Keys → Revoke
  • Keys can be scoped to read-only or specific resources

Error responses

// Invalid or missing key
{
  "error": "unauthorized",
  "message": "Invalid API key",
  "status": 401
}

Webhooks

Receive real-time notifications when events happen in heySynk.

Setup

Go to Admin Panel → Integrations → Webhooks and add your endpoint URL. We'll send a POST request to your URL for every event you subscribe to.

Events

  • conversation.created
  • conversation.resolved
  • conversation.assigned
  • message.created
  • contact.created
  • contact.updated
  • csat.submitted
  • campaign.triggered

Payload example

POST https://your-app.com/webhook

{
  "event": "conversation.resolved",
  "workspace_id": "ws_01HX...",
  "timestamp": "2026-04-02T11:00:00Z",
  "data": {
    "conversation_id": "conv_01HX...",
    "resolved_by": "agent_01HX...",
    "csat_score": 5
  }
}

Verifying signatures

Every webhook includes a X-HeySynk-Signature header — an HMAC-SHA256 of the payload using your webhook secret.

const crypto = require('crypto');
const sig = req.headers['x-heysynk-signature'];
const expected = crypto
  .createHmac('sha256', process.env.WEBHOOK_SECRET)
  .update(JSON.stringify(req.body))
  .digest('hex');
if (sig !== expected) return res.status(401).end();

Setup 5 min read

Getting started with heySynk in 5 minutes

Step 1 — Create your workspace

Go to app.heysynk.app and sign up. Choose a workspace slug — this becomes your login URL (e.g. acmecorp). You can change your display name and branding later in Admin Panel.

Step 2 — Invite your team

Go to Admin Panel → Agents → Invite Agent. Enter their email and assign a role (Agent, Senior Agent, Support Lead, or Admin). They'll get an invite email and can set their password.

Step 3 — Embed the chat widget

Go to Admin Panel → Chat Widget → Embed Code. Copy the one-line script and paste it before </body> on your website.

<script src="https://cdn.heysynk.app/widget.js" data-workspace="YOUR_SLUG"></script>

Step 4 — Set up your first routing rule

Go to Admin Panel → Routing Rules → New Rule. Start simple: assign all conversations to yourself. You can build more complex rules later.

Step 5 — Write your first KB article

Go to Knowledge Base → New Article. Write a quick FAQ — something like "How do I track my order?" The AI will use this when generating replies.

Home›Guides›AI Reply
AI Features

Using AI Reply effectively

Mira AI reads the full conversation and drafts a reply. Here's how to get the best results.

AI Reply vs AI Suggest

AI Reply — generates a complete reply ready to send. Click once to send or edit first.
AI Suggest — puts a draft in the reply box so you can refine before sending.

Tips for better replies

  • Keep your Knowledge Base articles up to date — AI uses them when generating replies
  • The more context in the thread, the better the reply. Don't delete old messages
  • If you edit a reply often, that topic probably needs a KB article
  • Use AI Suggest for sensitive conversations where you want full control
  • Set your workspace tone in Admin Panel → General → Tone (formal/friendly/neutral)

When NOT to use AI Reply

  • Very angry customers — always review first
  • Legal or compliance queries — human review required
  • Complex refund negotiations — needs human judgement

Admin

Setting up routing rules

How rules work

Rules run automatically on every new conversation. Each rule has conditions (when to fire) and actions (what to do). Rules are evaluated top-to-bottom — the first matching rule wins.

Available conditions

  • Visitor language — detected from browser locale
  • Page URL — contains, equals, starts with
  • Time of day — hour range, day of week
  • Is returning visitor — true/false
  • Visit count — greater than, equals
  • Channel — live chat, email, WhatsApp

Available actions

  • Assign to agent — specific agent or round-robin
  • Set priority — normal, high, urgent
  • Add tag — label conversations automatically
  • Send auto-reply — e.g. "We're closed, back at 9am"

Example: route by language

Rule: Arabic conversations → Fatima
Condition: Visitor language = Arabic
Action: Assign to Fatima Al-Rashid
Action: Add tag "arabic-support"
Home›Guides›Knowledge Base

Setting up your Help Centre

1. Create categories first

Go to Knowledge Base → Categories → New Category. Good examples: "Getting Started", "Orders & Shipping", "Returns & Refunds", "Account & Billing", "Technical Issues".

2. Write articles that actually help

  • Start with your 10 most-asked questions
  • Use clear headings — customers scan, they don't read
  • Add callout blocks for warnings and tips
  • Keep articles under 500 words where possible
  • Use AI Write → Improve to clean up rough drafts

3. Enable the public Help Centre

Go to Admin Panel → Help Centre → Enable. Set your custom domain (e.g. help.yourcompany.com), upload your logo, and set your brand colour. The Help Centre is now searchable by your customers.

4. Set SEO per article

Click the SEO tab in the article editor. Set a meta title (60 chars max) and description (160 chars max). Good SEO means customers find answers on Google before they ever write in.

Home›Guides›Campaigns

Creating your first campaign

Step 1 — Go to Campaigns → New Campaign

Give it a name (internal only), choose the message type: Chat Message, Site Banner, or Email.

Step 2 — Set your trigger rules

Add conditions using AND/OR logic. Example for a pricing page campaign:

URL contains /pricing
AND Time on page ≥ 30 seconds
AND Is returning visitor = true

Step 3 — Write your message

Keep it short. One question or one offer. You can use variables: {{visitor.name}}, {{visitor.company}}, {{page.title}}.

Step 4 — Set frequency

Choose how often the campaign fires per visitor: Once ever, Once per session, or Every visit. For most campaigns, "Once ever" is right — you don't want to spam.

Step 5 — Activate and monitor

Toggle the campaign to Active. Check Campaigns → Analytics after 24 hours to see Sent, Opened, Replied, and Conversion rate.

Home›Guides›WhatsApp

Connecting WhatsApp Business

Requirement: You need a WhatsApp Business Account (WABA) and a verified phone number. Personal WhatsApp accounts are not supported.

Step 1 — Set up WhatsApp Business API

Go to business.facebook.com → WhatsApp → Get Started. Follow Meta's verification process for your business and phone number. This takes 1–3 business days.

Step 2 — Get your API credentials

From your Meta Business Suite: note your Phone Number ID, WhatsApp Business Account ID, and generate a Permanent Access Token.

Step 3 — Connect in heySynk

Go to Admin Panel → Integrations → WhatsApp. Enter your Phone Number ID and Access Token. Click Connect. heySynk will configure the webhook automatically.

Step 4 — Test it

Send a WhatsApp message to your business number. It should appear in your heySynk inbox within seconds.
